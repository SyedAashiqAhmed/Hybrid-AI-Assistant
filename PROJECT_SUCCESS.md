## ğŸ¯ **Key Viva Points**

### **Q1: What is the hybrid approach?**
**A**: "We use rule-based keyword matching for system commands (flashlight, theme, URLs) and AI only for conversational queries. This saves cost, works offline, and is faster."

### **Q2: Why not use AI for everything?**
**A**: "Three reasons:
1. Cost - AI calls cost money, local execution is free
2. Speed - Local execution is instant, AI takes 2-3 seconds
3. Reliability - Works offline for system commands"

### **Q3: How does intent detection work?**
**A**: "The Intent Engine checks user input against predefined keywords. If it matches 'flashlight on', 'theme change', etc., it executes locally. Otherwise, it sends to Gemini AI."

### **Q4: What makes this project unique?**
**A**: "Most AI assistants send everything to AI. Our hybrid approach is smarter - we use the right tool for the right job. Simple commands run locally (free, fast, offline), complex queries use AI (intelligent, conversational)."

### **Q5: How do you ensure security?**
**A**: "Three layers:
1. URL whitelist - only approved domains
2. Input validation - all commands validated
3. API safety settings - Gemini blocks harmful content"

### **Q6: What are future enhancements?**
**A**: "Short-term: more system actions (Bluetooth, WiFi, volume). Long-term: offline AI using TensorFlow Lite, image recognition, wake word detection."
